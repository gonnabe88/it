# 정보기술부문 계획 추가
다음의 2개 테이블 변경사항을 반영하여 정보기술부문 계획를 관리하고 조회하는 화면을 만들어줘

## 업무 흐름
1. 정보기술부문 계획 > [계획 등록]
2. [계획 등록] 화면에서 대상년도와 신규/조정 여부를 선택하면 대상년도(PLN_YY)에 해당하는 사업년도(PRJ_YY)의 대상사업(프로젝트명) 목록을 조회
3. 대상사업(프로젝트명)을 선택 후 [생성] 버튼을 클릭하면 해당 사업들의 요약 및 세부정보를 화면에 표시
4. [계획 등록] 화면에서 [저장] 버튼 클릭 시 TAAABB_BPROJA(프로젝트관계) 및 TAAABB_BPLANM(정보기술부문 계획)에 저장하고 전체 프로젝트 내용을 JSON 형태로 TAAABB_BPLANM.PLN_DTL_CONE(계획세부내용)에 스냅샷 형태로 저장
5. [계획 조회] 화면에서 과거 계획 목록 조회
6. [계획 조회] 화면에서 특정 계획 선택 시 [계획 상세] 화면 이동

## [backend]
1. TAAABB_BPROJA(프로젝트관계) 테이블은 TAAABB_BPROJM(프로젝트 관리)와 TAAABB_BPLANM(정보기술부문 계획)을 N:N 매핑하는 테이블
2. TAAABB_BPROJA(프로젝트관계) 및 TAAABB_BPLANM(정보기술부문 계획) 관련 API 생성 필요

## [frontend]
1. 메뉴명 : 정보기술부문 계획
2. 화면구성 : 계획 등록, 계획 조회, 계획 상세
  1) 계획 등록 : 대상년도, 신규/조정 여부, 대상사업(프로젝트명) 선택 후 저장하는 화면
    - 대상사업(프로젝트명)은 TAAABB_BPROJM(프로젝트 관리)에서 조회
    - 다수의 대상사업(프로젝트명)을 선택하면 해당 사업들의 [예산 총계(총 예산, 자본예산, 일반관리비)], [부문(SVN_HDQ)별 사업목록], [사업유형(PRJ_TP)별 사업목록]을 화면에 표시
    - 저장 시 TAAABB_BPROJA(프로젝트관계) 및 TAAABB_BPLANM(정보기술부문 계획)에 저장하고 전체 프로젝트 내용을 JSON 형태로 TAAABB_BPLANM.PLN_DTL_CONE(계획세부내용)에 스냅샷 형태로 저장
  2) 계획 조회 : 등록된 계획을 데이터테이블 형태로 보여주는 화면
    - 테이블 레이아웃 : 대상연도, 신규/조정 여부, 계획명, 총 예산, 자본예산, 일반관리비
  3) 계획 상세 : 계획 조회화면에서 계획 선택 시 상세 정보를 보여주는 화면
    - [예산 총계(총 예산, 자본예산, 일반관리비)], [부문(SVN_HDQ)별 사업목록], [사업유형(PRJ_TP)별 사업목록]을 화면에 표시
    - 기준이 되는 데이터는 TAAABB_BPLANM.PLN_DTL_CONE(계획세부내용)을 파싱하여 화면에 표시
