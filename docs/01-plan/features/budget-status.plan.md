# budget-status Plan — 예산 현황 조회

## 1. 개요

### 1.1 배경 및 목적
현재 `/budget/list` 페이지는 편성요청 금액 기준의 통합 예산 목록을 제공하지만, 관리자가 편성률(BBUGTM.DUP_RT)을 적용한 **조정(편성) 금액**을 함께 확인할 수 있는 화면이 없다.

예산 편성 작업(`/budget/work`)에서 편성률을 적용한 후, 그 결과를 사업별 상세 수준에서 확인할 수 있는 **[예산 현황]** 메뉴를 신규 추가한다.

### 1.2 관련 테이블

| 테이블 | 역할 | 핵심 컬럼 |
|--------|------|-----------|
| TAAABB_BPROJM | 정보화사업 마스터 | PRJ_MNG_NO, PRJ_NM, PRJ_TP, SVN_HDQ, SVN_DPM 등 |
| TAAABB_BITEMM | 프로젝트 품목 | GCL_DTT(품목구분), GCL_AMT(금액), CUR(통화), GCL_QTT(수량), XCR(환산율) |
| TAAABB_BCOSTM | 전산업무비 마스터 | IT_MNGC_NO, ABUS_C, IOE_C, CTT_NM, IT_MNGC_BG, IT_MNGC_TP |
| TAAABB_BBUGTM | 예산 편성 결과 | ORC_TB(원본테이블), ORC_PK_VL(원본PK), IOE_C(비목코드), DUP_BG(편성예산), DUP_RT(편성률) |

### 1.3 데이터 흐름

```
편성요청 금액 (BITEMM.GCL_AMT × XCR / BCOSTM.IT_MNGC_BG × XCR)
    ↓ /budget/work에서 편성률 적용
조정(편성) 금액 (BBUGTM.DUP_BG = 요청금액 × DUP_RT / 100)
    ↓
[예산 현황] 화면에서 양쪽 금액을 나란히 표시
```

## 2. 요구사항

### 2.1 기능 요구사항

#### FR-01. [예산 현황] - [정보화사업] 탭 (ORN_YN = 'N')

BPROJM + BITEMM + BBUGTM을 조인하여 다음 컬럼을 표시한다:

**프로젝트 기본정보 (12개)**
- 프로젝트유형(PRJ_TP), 신규/계속(PUL_DTT), 사업명(PRJ_NM), 사업개요(PRJ_DES)
- 주관부문(SVN_HDQ), 주관부서(SVN_DPM), 주관팀 담당팀장(SVN_DPM_TLR), 주관팀 담당자(SVN_DPM_CGPR)
- IT담당부서(IT_DPM), IT담당팀장(IT_DPM_TLR), IT담당자(IT_DPM_CGPR)
- 추진가능성(PRJ_PUL_PTT), 시작일자(STT_DT), 종료일자(END_DT)

**편성요청 금액 (10개)**
- 개발비, 기계장치, 기타무형자산 → 자본예산(소계)
- 전산임차료, 전산여비, 전산용역비, 전산제비 → 일반관리비(소계)
- 총 사업예산(합계)

**기타 (2개)**
- 사전보고(RPR_STS), 전결권(EDRT)

**조정(편성) 금액 (10개)** — BBUGTM.IOE_C 기준 DUP_BG 매핑
- 개발비, 기계장치, 기타무형자산 → 자본예산(소계)
- 전산임차료, 전산여비, 전산용역비, 전산제비 → 일반관리비(소계)
- 총 사업예산(합계)

**합계: 약 34개 컬럼**

#### FR-02. [예산 현황] - [전산업무비] 탭

BCOSTM + BBUGTM을 조인하여 다음 컬럼을 표시한다:

**전산업무비 기본정보 (9개)**
- 신규/계속(PUL_DTT), 사업코드(ABUS_C), 비목코드(IOE_C)
- 담당부서(BICE_DPM), 담당팀(TEM), 계약명(CTT_NM), 계약업체명(CTT_OPP)
- 정보보호(INF_PRT_YN), 유형(IT_MNGC_TP)

**편성요청 금액 (5개)**
- 전산임차료, 전산여비, 전산용역비, 전산제비 → 일반관리비(합계)

**조정(편성) 금액 (5개)** — BBUGTM.IOE_C 기준 DUP_BG 매핑
- 전산임차료, 전산여비, 전산용역비, 전산제비 → 일반관리비(합계)

**합계: 약 19개 컬럼**

#### FR-03. [예산 현황] - [경상사업] 탭 (ORN_YN = 'Y')

BPROJM(경상) + BITEMM을 조인하여 다음 컬럼을 표시한다:

**경상사업 기본정보 (3개)**
- 신규/계속(PUL_DTT), 사업명(PRJ_NM), 사업개요(PRJ_DES)

**기계장치 상세 (5개)**
- 통화(CUR), 수량(GCL_QTT), 단가(금액/수량), 금액(GCL_AMT), 금액 환산(GCL_AMT × XCR)

**기타무형자산 상세 (5개)**
- 통화(CUR), 수량(GCL_QTT), 단가(금액/수량), 금액(GCL_AMT), 금액 환산(GCL_AMT × XCR)

**합계: 약 13개 컬럼**

#### FR-04. 컬럼 조정 기능
- 사용자가 표시할 컬럼을 선택/해제할 수 있는 UI 제공
- 체크박스 기반 컬럼 토글 (MultiSelect 또는 커스텀 패널)
- 선택 상태는 세션 내 유지 (localStorage 활용 검토)

#### FR-05. 표준 테이블 레이아웃
- 기존 `StyledDataTable` 컴포넌트를 사용
- 가로 스크롤 지원 (컬럼 수가 많으므로 `scrollable` + `scrollHeight` 설정)
- 금액 컬럼은 우측 정렬 + 천단위 콤마 포맷

#### FR-06. 백엔드 API
- **GET `/api/budget/status/projects`** — 정보화사업 + 편성금액 조회 (ORN_YN='N')
- **GET `/api/budget/status/costs`** — 전산업무비 + 편성금액 조회
- **GET `/api/budget/status/ordinary`** — 경상사업 품목 상세 조회 (ORN_YN='Y')
- 공통 파라미터: `bgYy` (예산년도)
- 백엔드에서 BBUGTM 조인하여 조정 금액을 계산해서 응답에 포함

## 4. 기술 설계 방향

### 4.1 백엔드

**신규 패키지**: `com.kdb.it.domain.budget.status`

| 클래스 | 역할 |
|--------|------|
| BudgetStatusController | REST 엔드포인트 3개 |
| BudgetStatusService | 비즈니스 로직 (조인 쿼리 + 금액 계산) |
| BudgetStatusDto | 응답 DTO (정보화사업/전산업무비/경상사업 각각) |
| BudgetStatusRepositoryCustom | QueryDSL 동적 쿼리 인터페이스 |
| BudgetStatusRepositoryImpl | QueryDSL 구현체 |

**쿼리 전략**:
- 정보화사업: BPROJM LEFT JOIN BITEMM (품목구분별 피벗) LEFT JOIN BBUGTM (비목별 피벗)
- 전산업무비: BCOSTM LEFT JOIN BBUGTM (비목별 매핑)
- 경상사업: BPROJM(ORN_YN='Y') LEFT JOIN BITEMM (기계장치/기타무형자산 분리)
- LEFT JOIN 사용 이유: 편성률 미적용 건도 편성요청 금액은 표시해야 함

### 4.2 프론트엔드

**신규 파일**:

| 파일 | 역할 |
|------|------|
| `pages/budget/status.vue` | 메인 페이지 (탭 3개, 컬럼 설정, 엑셀 다운로드) |
| `composables/useBudgetStatus.ts` | API 호출 composable |
| `types/budgetStatus.ts` | TypeScript 타입 정의 |

**컬럼 조정 기능**:
- PrimeVue `MultiSelect` 또는 커스텀 체크박스 패널
- `visibleColumns` ref로 표시 컬럼 관리
- `StyledDataTable`에 동적 `:columns` 바인딩
