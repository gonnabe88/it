# 다이어그램기능고도화 Plan — Tiptap 다이어그램 도구 고도화

## 1. 개요

### 1.1 배경 및 목적
현재 TiptapEditor.vue는 Excalidraw 하나만 지원한다. 업무 문서 작성 시 플로우차트(Diagrams.net)나 UI/설계 목업(Penpot) 수요가 있으나 에디터 외부에서 별도로 작업해야 한다.

PRD.md 요구사항에 따라 툴바 "다이어그램" 버튼을 드롭다운으로 확장하고, Diagrams.net과 Penpot을 iframe 임베드 방식으로 추가한다.

### 1.2 현행 구조
```
TiptapEditor.vue
  ├── 툴바 버튼: "다이어그램" (단일 클릭 → Excalidraw)
  ├── useExcalidrawDialog.ts  (다이얼로그 상태 singleton)
  ├── ExcalidrawWrapper.vue   (React 기반 Excalidraw CSR 래퍼)
  └── ExcalidrawNodeView.vue  (Tiptap NodeView: SVG 미리보기 + 편집/삭제)
```

### 1.3 로컬망 제약 사항 (추가 요구사항)

> **모든 기능은 외부 인터넷 없이 사내 로컬망에서 동작해야 한다.**

| 도구 | 로컬망 가능 여부 | 전제 조건 |
|------|--------------|---------|
| Excalidraw | **가능** (npm 패키지, 외부 의존 없음) | 없음 |
| Diagrams.net | **조건부 가능** | draw.io 오픈소스를 사내 서버에 자체 호스팅 필요 |
| Penpot | **조건부 가능** | Penpot 오픈소스를 사내 서버에 자체 호스팅 필요 |

**draw.io 자체 호스팅**: [jgraph/drawio](https://github.com/jgraph/drawio) — Apache 2.0 라이선스, Docker 이미지 제공 (`jgraph/drawio`)

### 1.4 도구별 특성 비교

| 항목 | Excalidraw | Diagrams.net (draw.io) | Penpot |
|------|-----------|----------------------|--------|
| 난이도 | 초급 | 중급 | 고급 |
| 호스팅 | 로컬 npm 패키지 | 사내 self-hosted draw.io | 사내 self-hosted Penpot |
| 연동 방식 | React 컴포넌트 직접 마운트 | iframe (자체 호스팅 URL) | iframe (자체 호스팅 URL) |
| 저장 형식 | SVG + JSON(sceneData) | SVG + XML | SVG |
| 재편집 | 가능 (sceneData 복원) | 가능 (XML 복원) | 가능 (SVG 재로드) |
| postMessage | 불필요 | 필요 (proto=json) | 필요 |
| 외부 인터넷 | 불필요 | 불필요 (self-hosted) | 불필요 (self-hosted) |

## 2. 요구사항

### 2.1 기능 요구사항

#### FR-01. 툴바 드롭다운 UI
- 기존 "다이어그램" 단일 버튼 → 클릭 시 드롭다운 메뉴 표시
- 드롭다운 항목: Excalidraw(초급), Diagrams.net(중급), Penpot(고급)
- 선택 시 해당 도구의 편집 다이얼로그 열기

#### FR-02. Excalidraw 기존 기능 유지
- 현행 Excalidraw 노드·NodeView·다이얼로그 변경 없음 (하위호환)
- 기존에 삽입된 Excalidraw 노드는 정상 렌더링·재편집 가능

#### FR-03. Diagrams.net (draw.io) iframe 연동
- **자체 호스팅 draw.io URL** 사용: `NUXT_PUBLIC_DRAWIO_URL` 환경변수로 관리 (예: `http://drawio.internal`)
- `{DRAWIO_URL}?embed=1&proto=json` 형식으로 iframe 열기
- postMessage `init` 이벤트 수신 후 편집 시작
- 저장 시 `export` 이벤트로 SVG + XML 수신
- Tiptap 커스텀 노드(`diagramsnet`)로 삽입 (속성: svgContent, xmlData)
- 재편집 시 xmlData를 iframe에 복원
- `NUXT_PUBLIC_DRAWIO_URL` 미설정 시 드롭다운에서 "(미설정)" 표시 및 비활성화

#### FR-04. Penpot iframe 연동
- Penpot 자체 호스팅 URL을 환경변수(`NUXT_PUBLIC_PENPOT_URL`)로 관리
- iframe으로 Penpot 열기, postMessage로 SVG 수신
- Tiptap 커스텀 노드(`penpot`)로 삽입 (속성: svgContent)
- 재편집 시 동일 iframe 재오픈 (SVG 복원)

#### FR-05. 공통 NodeView 패턴
- 각 도구별 NodeView: SVG 인라인 미리보기 + hover 편집/삭제 버튼 + 도구명 레이블
- 기존 ExcalidrawNodeView.vue와 동일한 UX 패턴 적용

### 2.2 비기능 요구사항

#### NFR-01. 보안
- iframe으로 삽입되는 SVG는 반드시 DOMPurify로 새니타이징 후 v-html 바인딩
- Penpot/Diagrams.net iframe의 postMessage origin 검증
- nuxt.config.ts의 CSP(Content Security Policy)에 iframe 허용 도메인 추가

#### NFR-02. 로컬망 전용 운영
- **외부 인터넷 의존 금지**: CDN, 외부 API, 외부 embed 서비스 사용 불가
- Excalidraw: npm 패키지 직접 사용 (이미 로컬) — 변경 없음
- draw.io URL: `NUXT_PUBLIC_DRAWIO_URL` 환경변수 필수 (미설정 시 비활성화)
- Penpot URL: `NUXT_PUBLIC_PENPOT_URL` 환경변수 (미설정 시 비활성화)
- 두 URL 모두 사내 IP/도메인이어야 함 (예: `http://drawio.internal`, `http://penpot.internal`)

#### NFR-03. CSR 전용
- Diagrams.net·Penpot iframe 컴포넌트는 `<ClientOnly>` 또는 `defineAsyncComponent` + `{ ssr: false }` 처리

#### NFR-04. 테스트
- 신규 composable(useDiagramsNetDialog, usePenpotDialog) 단위 테스트 작성
- 드롭다운 툴바 UI E2E 테스트(Playwright) 1개 이상

## 3. 설계 제약

### 3.1 기술 스택
- Framework: Nuxt 4 (Vue 3, `<script setup lang="ts">`)
- UI: PrimeVue + Tailwind CSS
- 에디터: Tiptap v2 (`@tiptap/vue-3`)
- 보안: `isomorphic-dompurify`

### 3.2 아키텍처 원칙
- 각 도구는 독립적인 Tiptap Node 확장 + NodeView 컴포넌트 + Dialog composable로 분리
- 도구별 컴포넌트 간 의존성 없음 (Excalidraw 수정 없음)
- postMessage origin 화이트리스트 검증 필수

### 3.3 Penpot 전제 조건
- Penpot은 자체 호스팅 인스턴스가 있어야 연동 가능
- `NUXT_PUBLIC_PENPOT_URL` 미설정 시 드롭다운에서 "(미설정)" 표시 및 클릭 비활성화
