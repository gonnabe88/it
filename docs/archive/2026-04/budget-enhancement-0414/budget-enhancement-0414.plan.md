# budget-enhancement-0414 Plan — 전산예산 시스템 개선

## 1. 개요

### 1.1 배경 및 목적
2026-04-14 PRD에 정의된 6개 요구사항을 구현한다. 결재 상신 간소화, 예산 편성률 개선, 예산현황 컬럼 변경, 전산업무비 금융정보단말기 관리 개선, 인라인 편집 UX 개선, 예산 신청 기간 제한 기능을 포함한다.

### 1.2 PRD 참조
- 원본: `PRD_20260414.md`

---

## 2. 요구사항 상세

### REQ-1. 결재 상신 전체 상신
- **화면**: `/budget/approval`
- **현재**: 체크박스로 신청서를 선택한 후 상신
- **변경**: 체크박스 제거, [결재 상신] 버튼 클릭 시 **미신청/반려 상태인 모든 신청서**를 일괄 상신
- **제외 대상**: 이미 '결재중' 또는 '결재완료' 상태인 신청서는 상신 대상에서 제외
- **영향 범위**:
  - Frontend: `it_frontend/app/pages/budget/approval.vue` — 체크박스 컬럼 제거, 상신 로직 변경
  - Backend: 기존 상신 API 수정 (선택된 ID 목록 대신 전체 미신청/반려 조회 후 상신)

### REQ-2. 예산 작업 편성률 개선
- **화면**: `/budget/work`
- **현재**: [변경비목 설정] 패널에서 비목별 편성률 설정
- **변경**:
  1. [변경비목 설정] 패널 제거
  2. [대상 목록] 테이블에 사업별 편성률 설정 컬럼 추가 (인라인 편집 가능)
  3. **기본값 로직** (공통코드 참조):
     - 자본예산(`IOE_CPIT`): PUL_DTT가 신규(`PUL_DTT_001`)이면서 금액 >= `DUP-AMT-300`(2억) → `DUP-IOE` 비목별 비율 적용 (기본 70%), 그 외 → 100%
     - 일반관리비(`IOE_IDR`, `IOE_LEAFE`, `IOE_SEVS`, `IOE_XPN`): `DUP-AMT-200`(기준액 0) 참조 → 사실상 항상 100% (단, `DUP-IOE` 비목별 비율이 설정된 경우 해당 값 적용)
- **공통코드 참조**:
  - `DUP-AMT-300`: 자본예산 편성기준액 (현재 200,000,000)
  - `DUP-AMT-200`: 일반관리비 편성기준액 (현재 0)
  - `DUP-IOE-*`: 비목별 편성 비율 (351=70%, 304=70%, 359=70%, 240=99%, 238=99%, 237=99%, 239=99%)
- **영향 범위**:
  - Frontend: `it_frontend/app/pages/budget/work.vue` — [변경비목 설정] 제거, [대상 목록]에 편성률 컬럼 추가
  - Backend: `BudgetWorkController`/`BudgetWorkService` — 편성률 기본값 계산 API, 편성률 저장 로직 수정

### REQ-3. 예산현황 컬럼 변경
- **화면**: `/budget/status` (전산업무비 탭)
- **현재**: [편성요청] / [조정(편성)] 컬럼 헤더
- **변경**:
  1. 컬럼 헤더를 [2025년] / [2026년]으로 변경
  2. 각 연도별 편성 금액 표시 (연도별 BCOSTM 데이터 기준)
- **영향 범위**:
  - Frontend: `it_frontend/app/pages/budget/status.vue` — 전산업무비 탭 컬럼 헤더 및 데이터 바인딩 변경
  - Backend: 전산업무비 조회 API 수정 — 연도 파라미터 기반 조회 또는 다년도 데이터 반환

### REQ-4. 전산업무비 금융정보단말기
- **화면**: `/info/cost`
- **변경**:
  1. 유형 컬럼 제거
  2. 금융정보단말기 여부 컬럼 추가 (체크박스 표시)
  3. 체크박스 체크 시 `/info/cost/terminal/form` 상세화면으로 이동하여 **신규 작성 가능**하도록 개선 (현재 신규 작성 불가)
  4. 상세화면에서 작성 완료 시 해당 탭을 닫고 전산업무비 목록 화면으로 복귀
- **영향 범위**:
  - Frontend: `it_frontend/app/pages/info/cost/index.vue` — 컬럼 변경
  - Frontend: `it_frontend/app/pages/info/cost/terminal/form.vue` — 신규 작성 모드 지원, 완료 시 탭 닫기 로직
  - Backend: `CostController`/`CostService` — 금융정보단말기 여부 필드 조회/저장 API

### REQ-5. 인라인 편집 개선
- **화면**: `/info/cost`, `/info/cost/terminal/form`
- **현재**: 테이블 인라인 편집 시 항상 편집 모드로 표시
- **변경**:
  1. 기본: 읽기 전용(view) 모드
  2. 컬럼 값 클릭 시 해당 셀만 편집 모드로 전환
  3. Enter 키 → 저장 (API 호출), Esc 키 → 취소 (원래 값 복원)
  4. 저장/취소 시 편집 모드 해제
- **영향 범위**:
  - Frontend: `it_frontend/app/pages/info/cost/index.vue` — 인라인 편집 컴포넌트 교체 또는 커스텀 로직 추가
  - Frontend: `it_frontend/app/pages/info/cost/terminal/form.vue` — 동일 인라인 편집 로직 적용
  - 공통 컴포넌트로 추출 가능: `InlineEditCell` 컴포넌트

### REQ-6. 예산 신청 기간 제한
- **화면**: `/budget` 및 하위 form 페이지 전체
- **공통코드**:
  - `BG-RQS-STA`: 예산 신청기간 시작일자 (현재: 2026-04-15)
  - `BG-RQS-END`: 예산 신청기간 종료일자 (현재: 2026-12-31)
- **변경**:
  1. **프론트엔드**: 기간 외 `/budget` 접속 시 안내 팝업(다이얼로그) 표시
  2. **프론트엔드**: 기간 외 [정보화사업], [전산업무비], [경상사업] 버튼 비활성화
  3. **프론트엔드**: 기간 외 form 화면 직접 접속 시 미들웨어에서 리다이렉트
  4. **백엔드**: 기간 외 예산 관련 생성/수정/삭제 API 호출 시 거부 (400 Bad Request)
- **영향 범위**:
  - Frontend: `it_frontend/app/pages/budget/index.vue` — 기간 체크 로직, 팝업, 버튼 비활성화
  - Frontend: 미들웨어 추가 — budget form 접근 시 기간 검증
  - Backend: 예산 관련 CUD API에 기간 검증 로직 추가 (공통코드 BG-RQS-STA/END 조회 후 비교)
  - Backend: 기간 조회 API (프론트엔드에서 사용)

## 3. 기술적 고려사항

### 3.1 공통코드 의존성
이번 작업에서 참조하는 공통코드가 많다. 프론트엔드에서 공통코드를 조회하는 기존 API(`/api/codes`)를 활용하되, 필요 시 그룹 조회를 추가한다.

| 공통코드 그룹 | 용도 | 참조 요구사항 |
|---------------|------|---------------|
| `DUP-AMT` | 편성기준액 | REQ-2 |
| `DUP_IOE` | 비목별 편성비율 | REQ-2 |
| `BG-RQS` | 예산 신청기간 | REQ-6 |

### 3.2 인라인 편집 컴포넌트
REQ-5의 인라인 편집은 `/info/cost`와 `/info/cost/terminal/form` 두 곳에서 동일하게 적용된다. 재사용 가능한 `InlineEditCell` 컴포넌트를 만들어 두 페이지에서 공유한다.

### 3.3 기간 제한 — 이중 검증
REQ-6은 프론트엔드와 백엔드 모두에서 기간 검증을 수행한다.
- **프론트엔드**: UX 차원의 안내 (팝업, 비활성화, 리다이렉트)
- **백엔드**: 보안 차원의 실제 차단 (API 레벨 검증). 프론트엔드 우회 방지

### 3.4 결재 상신 로직 변경
REQ-1에서 체크박스를 제거하므로, 백엔드 API의 요청 파라미터가 변경된다. 기존에 신청서 ID 목록을 받던 방식에서, 조건 기반 일괄 처리 방식으로 변경한다.
